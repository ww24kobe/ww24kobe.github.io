# 快速理解BFC

# 什么BFC

定义：块格式化上下文（Block Formatting Context，BFC），它是一个独立的渲染区域，只有Block-level box参与，它规定了内部的Block-level Box如何布局，并且这个区域与外部毫不相干。

也就是说触发了BFC(块格式化上下文（Block Formatting Context，BFC）)的容器就是页面上的一个完全隔离开的容器,容器里的子元素绝对不会影响到外面的元素。

下列方式会创建块格式化上下文

  • 浮动元素（元素的 float 不是 none）

  • 绝对定位元素（元素的 position 为 absolute 或 fixed）

  • 行内块元素（元素的 display 为 inline-block）

  • 表格单元格（元素的 display为 table-cell，HTML表格单元格默认为该值）

  • overflow 值不为 visible 的块元素

  • contain 值为 layout、content或 paint 的元素 弹性元素（display为 flex 或 inline-flex元素的直接子元素）

  • 网格元素（display为 grid 或 inline-grid 元素的直接子元素）

  • 等等

# 通过触发BFC解决的实际问题（作用）

    1. 浮动元素的父元素高度塌陷。
    1. 两栏自适应布局
    1. 外边距垂直方向重合

# 解决浮动元素的父元素高度塌陷

给父元素设置上面的任意一个属性即可触发BFC，如overflow:hidden。

这是因为父元素触发了BFC时，里面的子元素即使是浮动和定位也会参与高度计算,变相的实现了清除内部浮动的目的。

当然还有以下其他办法：

  • 给父元素设置float:left。优点：代码量少，缺点：影响之后的元素排列
  • 给父元素设置高度height。优点：没有学习成本。缺点：不灵活，难以维护，必须要知道子元素的高度。
  • 在父元素内容下面添加一个元素，设置属性为clear:both。优点：简单易懂，缺点：增加了无意义的标签。
  • 给最后一个浮动元素设置伪元素::after{clear:both}。 优点：结构和语义完全正确，缺点：复用不当导致代码量增加。

至于到底使用哪种方案，那还得根据实际需求来决定了。

# BFC总结

BFC就是页面上一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。 通常用来解决浮动和外边距合并等问题。因此，有个这个特性，我们布局的时候就可以防止一些意外情况的发生。
