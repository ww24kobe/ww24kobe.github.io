# requirejs模块化开发

# 什么是RequireJs

什么是模块? 一个文件就是一个模块。

RequireJS 是一个 JavaScript 模块加载器，使用 RequireJS 加载模块化脚本将提高代码的加载速度和质量。 官网地址：https://requirejs.org/docs/download.html

# Require优势

  • 声明不同js文件之间的依赖。
  • 异步“加载”。使用 RequireJS，会在相关的 js 加载之后再执行回调函数，这个过程是异步的，所以它不会阻塞页面。
  • 让我们的代码以模块化的方式去组织
  • ...

# 基本使用

1、在html文件中引入requirejs，,功能逻辑代码都在main.js中完成即可。

<script type="text/javascript" src="./require.js" data-main="./main.js"></script>

注意 script 标签，除了指定 require.js 路径外，还有个 data-main 属性：

2、在main.js文件中使用require.config()定义模块的别名和对应所在的路径

require.config({
	baseUrl:"./lib",  //相对于此路径去加载paths中的文件
	paths:{
         //引用名:'相对于baseUrl的路径
		jquery:"jquery",   
	}
});

说明：

  • baseUrl用于加载模块的根路径。在配置这个属性后，以后的文件都是在这个路径下去查找。

  • paths用于一些常用库或者文件夹路径映射，方便后面使用。js 文件的后缀可以省略）

3、使用require()函数指定依赖并实现功能代码 语法格式：

//指定依赖，实现功能
require(['依赖的模块'],function('引用名')){
	    //异步加载
	    //功能逻辑
})

# 定义模块

在require中可以使用require.define形式来定义我们的模块。 语法：

define([依赖的模块],function(){
    //模块代码
    //返回此模块中的代码
    return { };
});

第一个参数是数组，指定依赖的模块，如果没有依赖可留空数组或直接第二个function参数即可。 通过return返回一个对象向外暴露功能。
