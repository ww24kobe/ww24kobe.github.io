# NPM命令基本使用

# NPM介绍

1. NPM 是什么？

  • npm（全称Node Package Manager，即node包管理器）。

  • npm是Node.js默认的软件包管理器件。安装完毕node后，会默认安装npm

  • npm本身也是基于Node.js开发的包

  • 查看当前npm版本：npm -v

  • npm 官方网站，这是NPM 包管理库，存储了大量的JavaScript代码库

  • npm 官方文档，可以查看npm命令的具体使用

# NPM常用命令介绍

1、首先在项目根目录中先初始化项目描述文件package.json ，记录项目所依赖的包。

  • 可通过 npm init 命令根据引导创建 package.json 文件。
  • 或直接通过 npm init -y 或 npm init -yes 命令直接创建一个包含默认信息的package.json文件

2、npm下载模块

  • 在 https://www.npmjs.com/ 网站找到需要的包，进行下载

  • npm install xxx ：下载xxx模块到当前命令行所在的node_modules目录中，此目录若不存在则会自动创建。且安装的模块会写入package.json中的dependencies生成环境（线上环境）依赖中。

  • npm install xxx –-save ：下载并写入package.json中的dependencies生产环境依赖中，其中--save可简写-S

  • npm install xxx –-save-dev：下载并写入package.json中的devDependencies开发环境依赖中，其中--save-dev可简写为-D

  • npm install --production : 仅安装dependencies生产环境指定的依赖，

  • npm install 安装package.json文件中dependencies和devDependencies指定的所有模块

  • 在代码中通过 require('包名')加载对应的包会自动从node_modules中加载该模块

3、 npm 删除模块

npm uninstall xxx：删除本地xxx模块；

可以输入npm install --help查看详细用法

# package.json 文件中常见的选项

1、常见项目

  • name: 包的名称，由小写英文字母、数字和下划线组成，不能包含空格。

  • description：包的简要说明。

  • author：包的作者

  • version：符合语义化版本识别规范的版本

  • keywords：关键字数组，通常用于搜索。

  • repositories：仓库托管地址数组，每个元素要包含type（仓库的类型，如git）、url（仓库的地址）和path（相对于仓库的路径，可选）字段。

  • dependencies：记录项目在生产环境所依赖的包，一个关联数组，由包名称和版本号组成

  • devDependencies：记录项目在开发环境所依赖的包，一个关联数组，由包名称和版本号组成，一般记录第三方插件或工具包。

  • main： 当包被require时，指定包的入口js文件，即从这个js文件开始执行。

  • package.json文件说明：

  • package.json

  • Using a package.json

2、包版本号说明 在npm中安装固定的版本号package，只需要在其后加 @版本号

如：npm install xxx@1.2.2

  • 指定版本：比如1.2.2，遵循“大版本.次要版本.小版本”的格式规定，安装时只安装指定版本。
  • 波浪号+指定版本：比如~1.2.2，表示安装1.2.2以上的最新版本，但是不安装1.3.x的版本，也就是说安装时不改变大版本号和次要版本号。
  • 符号^+指定版本：比如ˆ1.2.2，表示安装1.2.2以上的最新版本，但是不安装2.x.x的版本，也就是说安装时不改变大版本号。
  • latest：安装最新版本。

# NPM 全局安装包

  • npm install 模块 -g :全局安装指的是把模块安装成了一个全局命令行工具。
  • 且全局安装的包不可以在代码中通过require() 的方式来加载。仅能当作一个全局的命令来使用

注意：全局安装仅能当全局命令去使用。

  • npm uninstall xxx -g：删除全局模块xxx；

# package.lock文件作用

锁定npm install所安装时的包的版本号，以保证其他人在npm install时,就能确保所有包与你上次安装的完全一样。

# nrm（npm下载源管理器）

nrm 是一个 NPM 源管理器，允许你快速地在如下 NPM 源间切换：

1、安装

npm install nrm -g

全局安装仅仅是当做一个全局的命令去使用。 2、使用列出可选的源 nrm ls

    nrm ls
  * npm ---- https://registry.npmjs.org/
  cnpm --- http://r.cnpmjs.org/
  taobao - https://registry.npm.taobao.org/
  nj ----- https://registry.nodejitsu.com/
  npmMirror  https://skimdb.npmjs.com/registry/
  edunpm - http://registry.enpmjs.org/
  带 * 的是当前使用的源，上面的输出表明当前源是官方源

3、切换到taobao源

nrm use taobao

4、测试 还可以通过 nrm test 测试相应源的响应时间

测试官方源的响应时间：nrm test npm

测试所有源的响应时间：nrm test

# 安装nodemon工具

每次修改完js文件之后，都要重新启动node服务，这样非常影响开发效率,使用nodemon工具会自动监测项目中的文件变化,一旦发现文件有改动,nodemon 会自动重启node服务

npm install nodemon -g

使用的时候，把node改为nodemon即可,如下

\Desktop\nodejs_ study>nodemon server3.js
